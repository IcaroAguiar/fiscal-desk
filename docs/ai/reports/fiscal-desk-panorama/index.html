Resumo executivo

Onde estamos

feito

Estamos no corte feat/fiscal-desk-local-base-prep, pronto para teste humano local antes de abrir PR/merge.

A base anterior já está em main: PR #1 fundação, PR #2 ledger/retomada, PR #3 histórico operacional, PR #4 entrega XLSX e PR #5 Base Pública Local operacional com fixture pequena. O corte atual amplia a PR #5: sai do provider local baseado apenas em fixture embutida e adiciona preparo real a partir de CSV local escolhido pelo usuário, com índice persistido no perfil do Electron.

O objetivo desta etapa não é baixar uma base pública real automaticamente. O desenho continua explícito e local-first: o usuário escolhe o CSV, o app prepara o índice local, mostra Data da Base, tamanho/uso estimado e só permite executar depois de base pronta e aceite.

Arquitetura

Fluxo do corte atual

parcial

O fluxo novo preserva a arquitetura porta + adapters:

  1. Renderer chama pickLocalPublicBaseSourceFile e prepareLocalPublicBase via preload.
  2. Main lê o CSV escolhido e delega para LocalPublicBaseStore.
  3. Store usa o preparador do core para normalizar CNPJs, validar flags simples_nacional/simei, deduplicar e persistir índice JSON atômico.
  4. app:get-defaults e local-public-base:get-status expõem o status para a UI.
  5. Ao processar CSV, main valida aceite + base pronta antes de ledger/provider.
  6. Provider base-publica-local consulta o índice preparado por SimplesLookupPort, sem Receita Web e sem consulta online por item.
