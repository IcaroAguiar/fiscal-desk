# Agentic Quality Gate Ratchet

This package was generated by `agentic-code-review/scripts/generate-quality-gate-ratchet.mjs`.

## Policy

- Keep new code clean first; do not require a big-bang cleanup of legacy code.
- Every PR publishes floating metrics in a Markdown table: large files, baseline, current value, target, delta, and status. Coverage and duplication are reported when the project starts producing those reports.
- Block regressions: secret leaks, failing native checks, duplication increases when a report exists, and large-file growth. Coverage is currently warning-only because the repository does not generate `coverage/lcov.info` yet. Independent agentic review remains required by the PR template for material PRs.
- Touched bad areas must improve or stay flat. If a PR touches an already-large file, the file must not grow without an explicit, reasoned, expiring exception in `quality-gate.config.json`.
- Baseline refresh is explicit and only allowed from a passing default-branch quality gate report.
- False positives and false negatives are tracked in `review-feedback.json`; false negatives produce an auto-improvement queue.
- Useful comments explain invariants, business rules, edge cases, and agent navigation context. Narrating syntax does not count as maintainability.

## Files

- `baseline.json`: current accepted state.
- `quality-gate.config.json`: thresholds and required checks.
- `check-ratchet.mjs`: CI checker and PR report generator.
- `comment-pr-report.mjs`: idempotent PR comment publisher for the generated Markdown table.
- `refresh-baseline.mjs`: controlled baseline refresh after a passing gate.
- `review-feedback.json`: optional human/reviewer feedback for false positives and false negatives.

## GitHub Actions

The active workflow is `.github/workflows/pr-quality-gate.yml`.

It runs:

- install with `pnpm install --frozen-lockfile`;
- `pnpm lint`;
- `pnpm typecheck`;
- `pnpm test`;
- `pnpm smoke:real-csv`;
- `pnpm smoke:visual`;
- `pnpm build`;
- `gitleaks/gitleaks-action@v2`;
- `node docs/ai/quality-gate/check-ratchet.mjs`;
- `node docs/ai/quality-gate/comment-pr-report.mjs` on pull requests.

The PR comment runs with `always()` so failures still leave an operational summary on the PR. It is limited to same-repository pull requests because fork PRs may receive a read-only `GITHUB_TOKEN`; fork runs still keep the uploaded quality artifacts. The comment is updated in place using the marker `<!-- fiscal-desk-quality-gate -->` on comments authored by GitHub Actions, so it should not create a new comment on every push or edit comments from humans.

This CI gate does not perform the independent agentic reviewer step. That remains a PR closeout requirement in `.github/pull_request_template.md` because Spark/executor agents are not valid reviewers.

When the agentic-review packet is not generated in CI, the quality table must not mark the agentic review row as `PASS`. It reports `NOT_ENFORCED` and keeps the independent reviewer as a PR closeout requirement.

## Feedback schema

`review-feedback.json` accepts:

```json
{
  "feedback": [
    {
      "case": "pr-123",
      "rule": "webhook-without-signature-verification",
      "outcome": "false-negative",
      "recommendation": "Add a detector for provider secret-token validation gaps."
    }
  ]
}
```

Use `false-positive`, `false-negative`, or `severity-mismatch` as outcomes.
